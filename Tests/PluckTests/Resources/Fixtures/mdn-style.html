Array.prototype.map()

The map() method of Array instances creates a new array populated with the results of calling a provided function on every element in the calling array. It is a non-mutating method that does not change the original array, but instead returns a brand-new array.

Syntax

map(callbackFn)
map(callbackFn, thisArg)

Parameters

The callbackFn is a function to execute for each element in the array. Its return value is added as a single element in the new array. The function is called with three arguments: the current element, its index, and the array map() was called on.

The optional thisArg is a value to use as this when executing callbackFn. See iterative methods for more information.

Description

The map() method is an iterative method. It calls a provided callbackFn function once for each element in an array and constructs a new array from the results. Read the iterative methods section for more information about how these methods work in general.

The callback function is invoked only for indexes of the array which have assigned values. It is not invoked for empty slots in sparse arrays, but it is invoked for indexes which have undefined as their value, or for any index whose value is otherwise considered undefined.

Since map builds a new array, calling it without using the returned array is an anti-pattern; use forEach or for...of instead. An indicator that you should not be using map: if you are not using the array it returns or you are not returning a value from the callback, then you should use forEach instead of map.
